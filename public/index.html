user routes

By log in you can add, edit and remove your own toys , Admin has the ability to delete and modify users and their toys.

domain https://api-chaya.onrender.com

#Method URL Body Description
GET /users/myInfo none Get your own info - by token.
GET /users/usersList none Get list of all users - by token admin.
POST /users 
{ 
fullName: {
firstName: Chaya,
lastName: Henner
},
"email": "chayahenner@gabay.com",
"password": "chaya1234"
}
Add a new user to the data base
POST (login) /users/login 
{
    "email": "chayahenner@gabay.com",
    "password": "chaya1234"
    }
User login, if the email and password are correct the server will respond with a token,
DELETE /users/{idDel} Delete a user from the data base. only admin token can delete every user.

Toys Routes

Some "GET" requests you can get information about the API, By adding these queries you can see the edit

  • Page : you can select the num of pages you want.

domain https://api-chaya.onrender.com/

#Method URL Body Description
GET /toys none Get a list of all the toys, will present 10 results per page.
GET /toys/search?s=doll none search for a toy by searching follwing name or info after the "s=", it will present 10 results per page.
GET /toys/prices?min=50&max=200 none Get a list of the toys by price min & max , it will present 10 results per page.
GET /toys/category/:cat none Get a list of all the toys in toys category, it will present 10 results per page.
POST /toys 

{
"name":"barbie Doll",
"info":"a girls dream doll",
"category":"Beauty",
"img_url":"Fun.jpg" ,
"price":83
}
Add a new toy to the data base, You need a token for this query ,
PUT /toys/ID 
 {
                                    "name":"barbie Doll",
                                    "info":"a girls dream doll",
                                    "category":"Beauty",
                                    "img_url":"Fun.jpg" ,
                                    "price":83
                                    }
Update a toy in the data base, only if the toy is yours.You need a token for this query.
DELETE /toys/ID none Delete a toy from the data base, only if the toy is yours. You need a token for this query.
